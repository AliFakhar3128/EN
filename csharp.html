خلاصه‌برگ جامع زبان برنامه‌نویسی C#

این خلاصه‌برگ به شما کمک می‌کند تا مفاهیم کلیدی زبان برنامه‌نویسی C# را از سطح مبتدی تا پیشرفته، به همراه توسعه برنامه‌های دسکتاپ، یاد بگیرید. مطالب به‌صورت دسته‌بندی‌شده، با مثال‌های عملی و نکات حرفه‌ای ارائه شده‌اند.

1. معرفی C#

تعریف

C# (سی‌شارپ) یک زبان برنامه‌نویسی شیءگرا، مدرن و چندمنظوره است که توسط مایکروسافت در سال 2000 به‌عنوان بخشی از چارچوب .NET معرفی شد. این زبان برای توسعه برنامه‌های دسکتاپ، وب، موبایل، بازی و اینترنت اشیا (IoT) استفاده می‌شود.

تاریخچه

  • 2000: معرفی C# توسط مایکروسافت با چارچوب .NET.
  • 2002-2003: استانداردسازی توسط Ecma و ISO/IEC.
  • 2024: آخرین نسخه، C# 13، با ویژگی‌های جدید مانند LINQ بهبودیافته و پشتیبانی از .NET 9.

ویژگی‌های کلیدی

  • تایپ ایمن: بررسی نوع داده‌ها در زمان کامپایل.
  • چندسکویی: اجرا روی ویندوز، لینوکس و macOS با .NET Core.
  • شیءگرا: پشتیبانی از مفاهیم کلاس، وراثت، چندریختی و کپسوله‌سازی.
  • اکوسیستم غنی: کتابخانه‌های گسترده در .NET.

2. مفاهیم پایه

سینتکس و ساختار

برنامه‌های C# معمولاً از ساختار زیر پیروی می‌کنند:

using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("سلام، دنیا!");
        }
    }
}

توضیحات:

  • using System: دسترسی به کتابخانه‌های اصلی .NET.
  • namespace: سازمان‌دهی کدها.
  • class: تعریف یک کلاس.
  • Main: نقطه ورود برنامه.
  • Console.WriteLine: نمایش خروجی در کنسول.

متغیرها و انواع داده‌ها

C# یک زبان تایپ ایمن است و متغیرها باید قبل از استفاده تعریف شوند.

نوع داده توضیحات مثال
int اعداد صحیح int age = 25;
double اعداد اعشاری double price = 19.99;
char کاراکتر تکی char grade = 'A';
string رشته متنی string name = "علی";
bool مقدار بولین (true/false) bool isActive = true;

مثال:

int myNum = 5;
double myDouble = 5.99;
char myLetter = 'D';
string myText = "سلام";
bool myBool = true;
Console.WriteLine($"{myNum}, {myDouble}, {myLetter}, {myText}, {myBool}");

عملگرها

  • حسابی: +, -, *, /, %
  • مقایسه‌ای: ==, !=, >, <, >=, <=
  • منطقی: &&, ||, !

مثال:

int a = 10, b = 5;
Console.WriteLine(a + b); // خروجی: 15
Console.WriteLine(a > b); // خروجی: true
Console.WriteLine(a == b); // خروجی: false

جریان کنترلی

  • شرطی (if): بررسی شرایط.
int x = 10;
if (x > 5)
{
    Console.WriteLine("x بزرگ‌تر از 5 است");
}
else
{
    Console.WriteLine("x کوچک‌تر یا برابر با 5 است");
}
  • سوئیچ (switch): انتخاب بین چندین گزینه.
int day = 3;
switch (day)
{
    case 1:
        Console.WriteLine("دوشنبه");
        break;
    case 3:
        Console.WriteLine("چهارشنبه");
        break;
    default:
        Console.WriteLine("روز دیگر");
        break;
}
  • حلقه‌ها:
    • for: برای تعداد مشخصی تکرار.
    for (int i = 0; i < 5; i++)
{
    Console.WriteLine(i);
}
    • while: تا زمانی که شرط برقرار است.
    int i = 0;
while (i < 5)
{
    Console.WriteLine(i);
    i++;
}

توابع (متدها)

متدها بلوک‌های کدی هستند که وظایف خاصی را انجام می‌دهند.

static int Add(int a, int b)
{
    return a + b;
}

static void Main(string[] args)
{
    int result = Add(3, 4);
    Console.WriteLine(result); // خروجی: 7
}

3. برنامه‌نویسی شیءگرا (OOP)

کلاس‌ها و اشیاء

کلاس قالب و شیء نمونه‌ای از کلاس است.

class Car
{
    public string Model;
    public int Year;

    public void Drive()
    {
        Console.WriteLine($"{Model} در حال حرکت است");
    }
}

static void Main(string[] args)
{
    Car myCar = new Car();
    myCar.Model = "پراید";
    myCar.Year = 1399;
    myCar.Drive(); // خروجی: پراید در حال حرکت است
}

وراثت

کلاس‌ها می‌توانند از کلاس‌های دیگر ارث ببرند.

class Vehicle
{
    public string Brand = "تویوتا";
}

class Car : Vehicle
{
    public string Model = "کمری";
}

static void Main(string[] args)
{
    Car myCar = new Car();
    Console.WriteLine($"{myCar.Brand} {myCar.Model}"); // خروجی: تویوتا کمری
}

چندریختی

امکان پیاده‌سازی متفاوت متدها در کلاس‌های مشتق‌شده.

class Animal
{
    public virtual void Speak()
    {
        Console.WriteLine("حیوان صدا می‌دهد");
    }
}

class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine("ووف!");
    }
}

static void Main(string[] args)
{
    Animal myDog = new Dog();
    myDog.Speak(); // خروجی: ووف!
}

کپسوله‌سازی

محدود کردن دسترسی به داده‌ها با استفاده از اصلاح‌کننده‌های دسترسی.

class Person
{
    private string name;

    public string Name
    {
        get { return name; }
        set { name = value; }
    }
}

static void Main(string[] args)
{
    Person person = new Person();
    person.Name = "علی";
    Console.WriteLine(person.Name); // خروجی: علی
}

انتزاع

استفاده از کلاس‌های انتزاعی برای تعریف رفتارهای مشترک.

abstract class Shape
{
    public abstract double Area();
}

class Circle : Shape
{
    public double Radius;

    public override double Area()
    {
        return Math.PI * Radius * Radius;
    }
}

static void Main(string[] args)
{
    Circle circle = new Circle { Radius = 5 };
    Console.WriteLine(circle.Area()); // خروجی: 78.54
}

4. موضوعات پیشرفته

ژنریک‌ها

ژنریک‌ها امکان نوشتن کدهای قابل استفاده مجدد و تایپ ایمن را فراهم می‌کنند.

class GenericList<T>
{
    private T[] items = new T[10];
    private int count = 0;

    public void Add(T item)
    {
        items[count] = item;
        count++;
    }
}

static void Main(string[] args)
{
    GenericList<int> numbers = new GenericList<int>();
    numbers.Add(1);
    numbers.Add(2);
}

دیلیگیت‌ها و رویدادها

دیلیگیت‌ها ارجاعاتی به متدها هستند و برای رویدادها استفاده می‌شوند.

public delegate void Notify();

class Process
{
    public event Notify ProcessCompleted;

    public void Start()
    {
        Console.WriteLine("پردازش شروع شد");
        ProcessCompleted?.Invoke();
    }
}

static void Main(string[] args)
{
    Process process = new Process();
    process.ProcessCompleted += () => Console.WriteLine("پردازش تمام شد");
    process.Start();
}

LINQ

LINQ برای پرس‌وجو از داده‌ها استفاده می‌شود.

int[] numbers = { 1, 2, 3, 4, 5 };
var evenNumbers = from num in numbers
                  where num % 2 == 0
                  select num;

foreach (var num in evenNumbers)
{
    Console.WriteLine(num); // خروجی: 2, 4
}

برنامه‌نویسی ناهمگام (Async/Await)

برای عملیات غیرهمزمان استفاده می‌شود.

public async Task<string> GetDataAsync()
{
    await Task.Delay(1000); // شبیه‌سازی عملیات زمان‌بر
    return "داده دریافت شد";
}

static async Task Main(string[] args)
{
    var result = await new Program().GetDataAsync();
    Console.WriteLine(result);
}

مدیریت خطاها

مدیریت خطاها با استفاده از بلوک‌های try-catch.

try
{
    int[] arr = { 1, 2, 3 };
    Console.WriteLine(arr[5]); // خطا: خارج از محدوده
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("خطا: " + ex.Message);
}
finally
{
    Console.WriteLine("بلوک finally اجرا شد");
}

5. چارچوب .NET و CLR

چارچوب .NET

.NET یک پلتفرم توسعه است که ابزارها و کتابخانه‌هایی برای ساخت برنامه‌ها فراهم می‌کند.

CLR (Common Language Runtime)

CLR محیط اجرایی .NET است که مدیریت حافظه، امنیت و اجرای کد را بر عهده دارد.

اسمبلی‌ها و فضای نام

  • اسمبلی: فایل‌های کامپایل‌شده (DLL یا EXE).
  • فضای نام: سازمان‌دهی کدها، مانند System, System.Collections.Generic.

6. توسعه برنامه‌های دسکتاپ

Windows Forms

چارچوبی ساده برای ساخت برنامه‌های دسکتاپ با رابط کاربری گرافیکی.

using System;
using System.Windows.Forms;

class Program
{
    static void Main()
    {
        Form form = new Form();
        Button button = new Button { Text = "کلیک کن", Location = new System.Drawing.Point(100, 100) };
        button.Click += (s, e) => MessageBox.Show("سلام، دنیا!");
        form.Controls.Add(button);
        Application.Run(form);
    }
}

WPF (Windows Presentation Foundation)

برای برنامه‌های پیشرفته‌تر با رابط کاربری مبتنی بر XAML.

<Window x:Class="WpfApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        Title="MainWindow" Height="350" Width="525">
    <Grid>
        <Button Content="کلیک کن" Click="Button_Click"/>
    </Grid>
</Window>

public partial class MainWindow : Window
{
    public MainWindow()
    {
        InitializeComponent();
    }

    private void Button_Click(object sender, RoutedEventArgs e)
    {
        MessageBox.Show("سلام، دنیا!");
    }
}

UWP (Universal Windows Platform)

برای برنامه‌های ویندوز 10 و بالاتر.

الگوی MVVM

الگوی Model-View-ViewModel برای جداسازی منطق برنامه از رابط کاربری.

8. بهترین روش‌ها و الگوهای طراحی

قراردادهای نام‌گذاری

  • کلاس‌ها و متدها: PascalCase (مثال: MyClass, MyMethod)
  • متغیرها: camelCase (مثال: myVariable)

سازمان‌دهی کد

  • متدهای کوتاه و متمرکز بنویسید.
  • از نام‌های معنادار استفاده کنید.

الگوهای طراحی

  • Singleton: اطمینان از وجود تنها یک نمونه از یک کلاس.
  • Factory: ایجاد اشیاء بدون مشخص کردن کلاس دقیق.
  • Observer: برای مدیریت رویدادها و اطلاع‌رسانی.

9. خطاهای رایج و دیباگ

خطا توضیح راه‌حل
NullReferenceException دسترسی به شیء null بررسی null با if یا عملگر ?.
IndexOutOfRangeException دسترسی به اندیس نامعتبر بررسی محدوده آرایه
InvalidCastException تبدیل نوع نامعتبر استفاده از as یا is

ابزارهای دیباگ:

  • نقاط توقف (Breakpoints): برای توقف اجرا در خطوط خاص.
  • پنجره‌های Watch و Immediate: برای بررسی مقادیر متغیرها.
