چیت‌شیت جامع پایتون

معرفی پایتون

پایتون یک زبان برنامه‌نویسی سطح بالا، تفسیری و داینامیک-تایپ است. کد پایتون سرآمد خوانایی و قالب‌بندی (Indentation) و سینتکس ساده دارد. این زبان برای طیف وسیعی از کاربردها – از اسکریپت‌های کوتاه گرفته تا برنامه‌های بزرگ و علم داده – مناسب است. پایتون به سرعت رشد کرده و همواره به‌روز می‌شود (نسخهٔ پایدار فعلی ۳٫۱۳/۳٫۱۴ است) و جامعۀ بسیار فعالی دارد.

مفاهیم پایه

  • متغیر و مقداردهی: در پایتون متغیرها نیازی به اعلام نوع داده ندارند و به‌سادگی با علامت = مقداردهی می‌شوند. به‌عنوان مثال: x = 10 یا name = "Ali". نوع متغیر بر اساس مقدار آن در زمان اجرا تعیین می‌شود (نوع‌دهی پویا).

  • انواع دادهٔ اولیه: پایتون چند نوع دادهٔ اصلی دارد:

    • int : اعداد صحیح با دقت نامحدود.
    • float : اعداد اعشاری (با دقت معین).
    • complex : اعداد مختلط (a+bj).
    • bool : نوع بولی (True/False) که زیرکلاسی از int است.
    • NoneType : شیء None برای نمایش نبود مقدار.

  • عملگرها: شامل + - * / // % ** برای عملیات ریاضی، == != < > <= >= برای مقایسه، and or not برای عملگرهای منطقی.

  • رشته (str): دنباله‌ای از نویسه‌ها است. می‌توان با کوتیشن‌های تکی، دوگانه یا سه‌تایی (برای چندخطی) ساخته شود. رشته‌ها در پایتون تغییرناپذیر (immutable) هستند.

  • تبدیل نوع (Casting): توابعی مثل int(x), float(x), str(x), bool(x) انواع داده را تبدیل می‌کنند.

سینتکس و قواعد اصلی

  • محدوده و تورفتگی: پایتون به‌جای آکولاد، از تورفتگی (Indentation) برای مشخص کردن بلوک‌های کد استفاده می‌کند. استاندارد تورفتگی، ۴ فاصله است.

  • دستورات شرطی:

    if شرط:
    # بلوک اجرا در صورت درست بودن شرط
elif شرط2:
    # بلوک دوم
else:
    # بلوک در صورت عدم برقرار بودن شرایط

  • حلقه‌ها:

    • for برای تکرار روی دنباله‌ها:

      for i in range(5):
    print(i)

    • while برای تکرار تا زمانی که شرط برقرار است:

      while condition:
    # انجام کاری
    if break_condition:
        break

    • break برای خروج از حلقه، continue برای عبور از یک تکرار بدون ادامه بقیه بدنه، و pass برای جایگزینی بلاک خالی است.

    • عبارت match-case: در نسخه‌های اخیر (۳٫۱۰+) امکان الگویابی ساختاری با match ... case اضافه شده که معادل سوئیچ پیچیده‌تری است.

  • توابع (functions): با کلمه‌کلیدی def تعریف می‌شوند:

    def my_function(arg1, arg2=5, *args, **kwargs):
    """توضیح کوتاه تابع (Docstring)."""
    return arg1 + arg2

    پارامترهای پیش‌فرض، موقعیتی (positional)، نام‌دار (keyword)، و آرگومان متغیر (*args, **kwargs) پشتیبانی می‌شوند. تابع می‌تواند مقدار بازگشتی داشته باشد (return).

  • عبارات مختصر (lambda): یک تابع ناشناس تک‌خطی با lambda تعریف می‌شود:

    square = lambda x: x**2
print(square(5))  # خروجی: 25

  • Scoping و نامگذاری: اسکوپ‌های (محدوده‌های) متغیر شامل local, global, و built-in هستند. برای دسترسی یا تغییر متغیر بیرونی در تابع، از global یا nonlocal استفاده می‌شود. تابع globals() و locals() برای مشاهدهٔ اسکوپ‌ها کاربرد دارند.

  • کامنت: با # نوشته می‌شود و تا پایان خط ادامه دارد.

ساختارهای داده

پایتون ساختارهای داده قدرتمندی دارد که اکثر آنها در کتابخانۀ استاندارد به صورت شیء توکار هستند:

  • لیست (list): دنبالهٔ تغییرپذیر (mutable) از مقادیر با امکان تکرار. روش‌های مهم آن شامل append(), insert(), remove(), pop(), sort(), reverse(), count(), index(), clear(), copy() و غیره است. لیست با [] یا تابع list() ساخته می‌شود.
    مثال: 

    nums = [1, 2, 3]
nums.append(4)    # [1, 2, 3, 4]
nums[1] = 10      # [1, 10, 3, 4]
x = nums.pop()    # x=4, nums=[1, 10, 3]

  • تاپل (tuple): دنبالهٔ تغییرناپذیر (immutable). تاپل‌ها می‌توانند به عنوان کلید در دیکشنری یا عناصر مجموعه استفاده شوند (اگر شامل عناصر متغیر نباشند). با () یا تنها با کاما تعریف می‌شوند.

  • رشته (str): دنبالهٔ تغییرناپذیر از نویسه‌ها (متنی). از عملگرهای + (الحاق)، * (تکرار)، برش (s[i:j]) و توابع فراخوانی len(), upper(), strip() و غیره پشتیبانی می‌کند.

  • مجموعه (set): مجموعه‌ای نامرتب و بدون تکرار از عناصر هَش‌پذیر (immutable). عملیات‌هایی مثل اجتماع (|)، اشتراک (&)، اختلاف (-)،‌ زیرمجموعه/ابرمجموعه پشتیبانی می‌شود. مثال:

    a = {1, 2, 3}
b = set([2, 3, 4])
a.add(5); a.remove(1)
c = a & b  # اشتراک: {2, 3}

    عبارت مجموعه: مشابه لیست‌کامل می‌توان مجموعه کامل نوشت: {x for x in iterable if condition}.

  • دیکشنری (dict): مجموعه‌ای از جفت‌های کلید:مقدار بدون ترتیب مشخص. کلیدها باید یکتا و از نوع immutable باشند (مثلاً str, int, یا tuple تو در تو از انواع immutable). مثال: 

    tel = {'jack': 4098, 'sape': 4139}  # تعریف دیکشنری با {}
tel['guido'] = 4127                # افزودن یا بروزرسانی مقدار
value = tel['jack']                # دسترسی: مقدار 4098 را می‌دهد
del tel['sape']                    # حذف کلید و مقدار

    از dict() یا استفاده از دیکشنری‌کامل ({k: v for ...}) هم می‌توان دیکشنری ساخت. مثلاً {x: x**2 for x in (2, 4, 6)} خروجی {2:4, 4:16, 6:36} تولید می‌کند.

  • None: مقدار خاص نمایانگر نبود مقدار است. مثلاً تابعی که مقدار صریح برنمی‌گرداند به‌طور پیش‌فرض None برمی‌گرداند.

  • نوع باینری: bytes (غیرقابل تغییر) و bytearray (قابل تغییر) برای داده‌های باینری استفاده می‌شوند.

  • عبارات درک (Comprehensions): برای تولید سریع داده‌ها کاربرد دارند. به‌عنوان مثال [x**2 for x in range(10)] لیستی از مجذورات ۰ تا ۹ می‌سازد. به همین شکل می‌توان {expr for ...}, {key: value for ...} برای مجموعه و دیکشنری استفاده کرد.

عملیات ورودی/خروجی و فایل

  • چاپ در خروجی: تابع print() برای چاپ رشته یا مقادیر استفاده می‌شود. (مثلاً print("Hello", name)).

  • ورودی از کاربر: تابع input() برای خواندن خط ورودی استفاده می‌شود (همواره خروجی آن رشته است).

  • فایل‌ها: برای کار با فایل از تابع open() استفاده می‌شود. به‌صورت کلی: 

    f = open('file.txt', mode='r', encoding='utf-8')
text = f.read()   # خواندن کل فایل به عنوان رشته
f.close()

    حالت‌های رایج: 'r' (خواندن)، 'w' (نوشتن/بازنویسی)، 'a' (افزودن به انتها)، و افزودن 'b' برای حالت باینری. توصیه: از عبارت with استفاده کنید تا فایل خودکار بسته شود: 

    with open('data.txt', 'r', encoding='utf-8') as f:
    contents = f.read()
# در اینجا فایل بسته شده است

  • خواندن/نوشتن خطی: متدهای f.readline() و f.readlines() یا پیمایش روی شیء فایل برای خواندن خط به خط. برای نوشتن، از f.write(str) استفاده می‌شود.

  • JSON: برای تبادل داده‌ی ساختاری، ماژول json کاربرد دارد. تابع json.dumps(obj) شیٔ پایتون را به رشتهٔ JSON تبدیل می‌کند و json.loads(s) معکوس آن را انجام می‌دهد. مثال: 

    import json
data = {'name': 'Ali', 'age': 30}
s = json.dumps(data)         # '{"name": "Ali", "age": 30}'
obj = json.loads(s)         # {'name': 'Ali', 'age': 30}

  • پردازش فایل‌های بزرگ: هنگام پردازش فایل‌های حجیم، از خواندن بخش به بخش (مثلاً در حلقه با for line in file) برای کارایی حافظه استفاده کنید.

کنترل خطاها و استثناها

  • خطاهای نحوی (SyntaxError): اشتباهات نگارشی (مانند فراموش کردن : در انتهای if یا {} نامتعادل). مثلاً نوشته شدن while True print('Hello') باعث SyntaxError: invalid syntax می‌شود.

  • استثناها (Exceptions): هنگام اجرا ممکن خطاهایی مانند تقسیم بر صفر (ZeroDivisionError)، استفاده از نام تعریف نشده (NameError) یا اشتباه نوع داده (TypeError) رخ دهد. پایتون وقتی با استثنایی روبه‌رو می‌شود، پیغام‌ خطای تفصیلی نمایش می‌دهد.

  • مدیریت استثنا: با بلوک try/except می‌توان از قطع اجرای برنامه جلوگیری کرد. ساختار کلی:

    try:
    # انجام کاری که ممکن است خطا دهد
except TypeError:
    # رسیدگی به خطای TypeError
except (ValueError, KeyError):
    # چند نوع خطا
else:
    # اگر خطایی رخ نداد
finally:
    # این بخش همیشه اجرا می‌شود (برای آزادسازی منابع)

    مثال:

    try:
    num = int(input("Enter number: "))
except ValueError:
    print("عدد نامعتبر است. دوباره تلاش کنید.")
else:
    print("عدد وارد شده:", num)

  • ایجاد (برانگیختن) استثنا: با دستور raise می‌توان به‌صورت دستوری خطا ایجاد کرد. مثلاً raise ValueError("پیام خطا"). همچنین می‌توان یک استثنای قبلی را مجدداً پرتاب کرد یا از raise ... from ... برای زنجیره کردن استثناها استفاده نمود.

  • تعریف استثنای جدید: برای تعریف کلاس استثنا اختصاصی، کافی است یک کلاس جدید از Exception ارث‌بری کند: 

    class MyError(Exception):
    pass

raise MyError("Something went wrong")

  • استثنای نهایی: بلوک finally (در try/except/finally) تضمین می‌کند که مقداری از کد حتما اجرا شود، حتی در صورت وقوع استثنا. مثال:

    try:
    value = data[i]
except IndexError:
    print("اندیس نامعتبر!")
finally:
    print("پایان دسترسی")

ماژول‌ها و بسته‌ها

  • ماژول: فایل پایتون با پسوند .py است که تعاریف توابع، کلاس‌ها و متغیرها را در خود دارد. برای استفاده از ماژول در برنامه، از import استفاده می‌شود. مثلاً اگر فایلی به نام fibo.py داشته باشیم:

    import fibo
fibo.fib(10)      # فراخوانی تابع fib از ماژول fibo

    این کار، نام ماژول (fibo) را به فضا (namespace) اضافه می‌کند و با fibo.name می‌توان به موارد آن دسترسی داشت.

  • عبارت‌های import:

    • import module کل ماژول را وارد می‌کند.
    • from module import name فقط نام (تابع/کلاس) خاصی را وارد می‌کند (بدون نیاز به پیشوند ماژول).
    • from module import * همهٔ نام‌های ماژول را وارد می‌کند (معمولاً توصیه نمی‌شود چون خوانایی را کاهش می‌دهد).
    • می‌توان با as نام مستعار تعیین کرد: import numpy as np.

  • پکیج (Package): راهی برای سازماندهی ماژول‌ها در دایرکتوری‌های تو در توست. هر پوشه پکیج باید فایل __init__.py (حتی خالی) داشته باشد تا شناخته شود. برای مثال ساختار:

    sound/
    __init__.py
    effects/
        __init__.py
        echo.py
        surround.py
    filters/
        __init__.py
        equalizer.py

    برای import:

    import sound.effects.echo
sound.effects.echo.echofilter(...)
from sound.effects import echo
echo.echofilter(...)
from sound.effects.echo import echofilter
echofilter(...)

    برای جزئیات بیشتر به بخش بسته‌ها در مستندات پایتون مراجعه کنید.

  • کتابخانه استاندارد: پایتون کتابخانه استاندارد گسترده‌ای دارد شامل ماژول‌های مفید. به عنوان نمونه:

    • os, sys برای کار با سیستم عامل و پارامترهای محیطی.
    • math برای توابع ریاضی، random برای تولید اعداد تصادفی.
    • datetime, time برای تاریخ/زمان.
    • re برای عبارات منظم (regex).
    • json برای پردازش JSON.
    • logging برای گزارش‌گیری.
    • itertools، collections، functools و … برای ابزارهای پیشرفته.
    • subprocess برای اجرای فرایندهای خارجی.
      استفاده از import math; math.sqrt(16) و مشابه آن رایج است.

  • نصب بسته‌های خارجی (پکیج‌ها): ابزار pip برای نصب بسته‌های سوم‌شخص از PyPI به‌کار می‌رود. مثلا:

    pip install requests

    توصیه می‌شود در هر پروژه یک محیط مجازی (Virtual Environment) بسازید. ماژول venv با دستور python -m venv env محیط مجازی ایجاد می‌کند. این محیط حاوی نسخهٔ مجزا از پایتون و بسته‌ها است تا تداخل بین پروژه‌ها رخ ندهد. پس از ساخت، باید محیط را فعال (activate) کنید تا دستورات pip و python در آن محیط اجرا شوند.

برنامه‌نویسی پیشرفته

  • OOP – کلا‌س‌ها:

    class MyClass(BaseClass):
    class_var = 0       # متغیر کلاس
    def __init__(self, x):
        self.x = x      # متغیر نمونه (شیء)
    def method(self):
        return self.x

    – کلاس‌ها بسته‌ای از داده (متغیرها) و روش‌ها (متدها) هستند. متدها پارامتر اول self دارند که مرجع به نمونهٔ جاری است. ارث‌بری (Inheritance) پایتون امکان چند پایه (Multiple Inheritance) را می‌دهد. می‌توان متدها و سازنده (__init__) را بازنویسی و متد والد را با super() فراخوانی کرد. ویژگی‌هایی مثل خصوصی‌سازی مطلق وجود ندارد، اما قراردادی پیشوند _name یا __name برای خصوصی کردن استفاده می‌شود.

  • متدهای خاص (Magic Methods): تعاریف مانند __str__, __repr__ برای نمایش متناظر شیء، __len__, __add__ و... برای اپراتورهای خاص.

  • توابع لامبدا پیشرفته: علاوه بر lambda ساده، کتابخانه functools ابزارهایی مثل partial و reduce دارد.

  • عبارات درک پیشرفته: لیست/دیکشنری/مجموعه-کامل برای فیلتر و تولید داده‌ها. مثال: [x*2 for x in vec if x>=0] یک لیست از مقادیر غیرمنفی است.

  • تولیدکننده‌ها (Generators): تابعی که yield دارد یک تولیدکننده می‌سازد. مقادیر را به تدریج تولید کرده و حالت اجرای تابع را حفظ می‌کند. مثال:

    def countdown(n):
    while n > 0:
        yield n
        n -= 1

for i in countdown(5):
    print(i)  # 5 4 3 2 1

    عبارات توليد با پرانتز (نه کروشه) مانند (x*x for x in range(10)) نیز وجود دارند که تولیدکننده می‌سازند.

  • Context Manager: برای آزادسازی خودکار منابع از دستور with استفاده می‌شود (مثلاً در فایل، with open(...) as f:). می‌توان با تعریف متدهای __enter__ و __exit__ کلاس خودمان را به عنوان مدیجر تعریف کرد.

  • دیگر ابزار پیشرفته:

    • ماژول itertools: توابع کارآمد برای کار با تکرارها (مثل chain, cycle, combinations, permutations و...).

    • تست و اشکال‌زدایی: کتابخانه‌هایی مثل unittest، pytest برای تست واحد. ماژول pdb برای Debugging.

    • ساخت بسته: برای انتشار کد، از setup.py یا ابزارهای جدیدتر مثل Poetry, setuptools استفاده می‌شود.

    • Type Hints (تایپ‌هینت): از نسخه‌های جدید، می‌توان نوع ورودی/خروجی توابع را با anotations مشخص کرد. PEP 484 استاندارد آنهاست (مثلاً def f(x: int) -> str:).

    • چندپردازشی و چندریسمانی:

      • ماژول threading برای Thread (رشته) و multiprocessing برای Process (فرآیند) وجود دارد. به دلیل GIL، نخ‌ها برای کارهای I/O-bound مفیدند و فرآیندها برای کار CPU-bound. مثال threading.Thread(target=func).start() یا کلاس Process در multiprocessing.
      • asyncio و async/await: برای برنامه‌نویسی همزمان بدون نخ. با async def و await نوشته می‌شود. کتابخانۀ asyncio برای IO همزمان و شکل دادن event loop به‌کار می‌رود.

نکات حرفه‌ای و بهینه‌نویسی

  • لیست‌کامل و توابع Built-in: استفاده از [x*2 for x in vec] سریع‌تر و پایتونیک‌تر از حلقهٔ for معادل است. از توابع داخلی مانند map(), filter(), zip(), enumerate(), sorted(), sum(), any(), all() برای کد خواناتر و سریع‌تر بهره ببرید.
  • کدی پایتونیک: قانون «احتمالا بارها استفاده کن، نه تصدیق پیش از عمل» (EAFP) را رعایت کنید. مثلاً به‌جای بررسی وجود کلید در دیکشنری قبل از گرفتن آن، از try/except KeyError استفاده کنید یا get().
  • زمینه (Context): همه کلاس‌ها و کتابخانه‌ها به دنبال پایتون «داينامیک» هستند، پس مفاهیمی مانند متا‌کلاس‌ها و تغییر ساختار در زمان اجرا وجود دارد اما کمتر استفاده مستقیم می‌شود.
  • جمع‌بندی: برای قابل نگهداری بودن کد، استاندارد PEP8 را رعایت کنید (چهار فاصله تورفتگی، طول مناسب خطوط، نام‌گذاری معنادار). اسناد توصیفی (Docstrings) بنویسید. از مدیریت نسخه (Git) و محیط مجازی استفاده کنید. از ابزارهایی مانند linter و formatter (مثلاً flake8, black) بهره ببرید.

استفاده در پروژه‌های واقعی

  • خودکارسازی (Scripting): پایتون برای اسکریپت‌های خودکارسازی فایل، پردازش متن (مثلاً با re), کار با سیستم (ماژول os, subprocess) فوق‌العاده مناسب است.

  • وب (Web): با فریم‌ورک‌هایی مانند Django, Flask, FastAPI برنامه‌های وب می‌سازند. این فریم‌ورک‌ها امکانات مسیریابی، ORM و … ارائه می‌کنند.

  • علم داده و یادگیری ماشین: کتابخانه‌های پرطرفدار NumPy, Pandas, Matplotlib, Scikit-learn در اکوسیستم پایتون توسعه داده شده‌اند. برای کار با داده‌ها، تحلیل آماری و مدل‌سازی استفاده می‌شوند.

  • توسعه‌ی رابط گرافیکی: کتابخانه‌هایی مثل Tkinter (کلاسیک)، PyQt یا Kivy برای ساخت رابط‌های کاربری دسکتاپ وجود دارند.

  • اتوماسیون و وب‌سرویس: پکیج‌هایی مانند requests برای ارسال HTTP و BeautifulSoup یا Scrapy برای وب‌خراشی استفاده می‌شوند.

  • مثال ساده: خواندن یک فایل CSV و پردازش:

    import csv

with open('data.csv', newline='') as csvfile:
    reader = csv.DictReader(csvfile)
    for row in reader:
        print(row['name'], row['value'])

  • نسخهٔ پایتون: در پروژه‌ها معمولاً از پایتون ۳ استفاده کنید (پایتون ۲ منسوخ است). مطمئن شوید نسخه‌(های) مورد نیاز را در مستندات پروژه ذکر کنید.

خطاهای رایج و رفع آنها

  • IndentationError: خطاهای تورفتگی؛ معمولاً ناشی از ترکیب تب و فاصله یا تورفتگی نادرست است. ثابت‌نگه‌داشتن قالب (مثلاً فقط فاصله) ضروری است.
  • NameError: استفاده از نامی که تعریف نشده (spam = 3; print(spamm) خطا می‌دهد). بررسی کنید نام متغیر قبل از استفاده تعریف شده باشد.
  • TypeError/ValueError: ورودی نادرست به تابع یا عملیات؛ برای مثال جمع رشته و عدد یا int("abc"). معمولاً با تبدیل نوع مناسب یا چک کردن قبل از تبدیل رفع می‌شود.
  • IndexError/KeyError: دستیابی به اندیس ناموجود لیست یا کلید غیرموجود دیکشنری. با استفاده از تابع in یا متد get() برای دیکشنری، یا چک طول لیست، می‌توان از وقوع آن جلوگیری کرد.
  • SyntaxError: فراموش کردن :, پرانتز یا خطای نگارشی. خط پیام خطا جای تقریبی مشکل را نشان می‌دهد.
  • ModuleNotFoundError: عدم نصب یا شناخته نشدن ماژول. اطمینان حاصل کنید پکیج مورد نیاز نصب شده (با pip) و نام ماژول صحیح باشد.
  • Others: هرازگاهی خطاهای ناشی از نسخهٔ پایتون (مثلاً استفاده از ویژگی‌های جدید در نسخه قدیمی) پیش می‌آید؛ مستندات نسخه را چک کنید.

منابع یادگیری و پیشرفت

  • مستندات رسمی: مستندات پایتون (Tutorial و Library Reference) در python.org جامع‌ترین منبع هستند. همیشه به آخرین نسخهٔ پایتون مراجعه کنید.
  • PEP‌ها: پیاده‌سازی استانداردها و ویژگی‌های پایتون در مستندات PEP (Python Enhancement Proposals) آمده است. مثلاً PEP 8 برای سبک کدنویسی.
  • کتاب‌ها و آموزش‌ها: «راهنمای جامع پایتون» و «یادگیری پایتون» از منابع خوب فارسی/انگلیسی هستند. سایت‌هایی مانند Real Python، GeeksforGeeks و مستندات PyMOTW (Python Module of the Week) آموزش‌های خوبی دارند.
  • جامعه: با شرکت در انجمن‌ها و گروه‌های برنامه‌نویسی (مثل Stack Overflow) می‌توانید مشکلات را سریع‌تر رفع کنید.
  • تمرین: پس از مطالعه، پروژه‌های کوچک یا تمرین‌های حل شده را خودتان اجرا کنید تا دستتان راه بیفتد. ساخت پروژه‌های ساده (وب‌سایت کوچک، اسکریپت پردازش داده، بازی کوچک و ...) مهارت شما را سریع‌تر می‌کند.

تذکر: همواره کد خود را تمیز و خوانا نگه دارید و از ابزارهای مدیریت بسته (pip, venv) استفاده کنید. پیروی از سبک استاندارد و یادگیری مداوم، کلید بهره‌وری بالا در پایتون است.
