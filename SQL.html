چیت شیت جامع SQL

1. مبانی و اصول اولیه

SQL چیست؟

SQL (زبان پرس‌وجوی ساختاریافته) یک زبان برنامه‌نویسی است که برای مدیریت و دستکاری پایگاه‌های داده رابطه‌ای استفاده می‌شود. این زبان به کاربران اجازه می‌دهد تا داده‌ها را در پایگاه داده ایجاد، خواندن، به‌روزرسانی و حذف کنند.

تاریخچه و تکامل SQL

SQL در دهه 1970 توسط محققان IBM توسعه یافت و به یک زبان استاندارد برای پایگاه‌های داده رابطه‌ای تبدیل شد. استانداردهای مختلفی مانند SQL-86، SQL-92 و SQL:1999 در طول زمان معرفی شده‌اند که قابلیت‌های جدیدی را به این زبان اضافه کرده‌اند.

گویش‌های مختلف SQL

چندین گویش SQL وجود دارد، از جمله:

  • MySQL: محبوب برای برنامه‌های وب.
  • PostgreSQL: قدرتمند برای برنامه‌های پیچیده.
  • SQL Server: مناسب برای محیط‌های مایکروسافت.
  • Oracle: برای پایگاه‌های داده سازمانی.
  • SQLite: سبک و مناسب برای برنامه‌های کوچک.

هر گویش ممکن است تفاوت‌های جزئی در نحو و ویژگی‌ها داشته باشد.

راه‌اندازی محیط SQL

برای شروع کار با SQL، به یک سیستم مدیریت پایگاه داده (DBMS) نیاز دارید. می‌توانید نرم‌افزارهایی مانند MySQL Workbench، pgAdmin یا SQL Server Management Studio را نصب کنید یا از ابزارهای آنلاین مانند SQL Fiddle استفاده کنید.

2. نحو و معانی

انواع داده در SQL

انواع داده‌های رایج در SQL شامل موارد زیر هستند:

  • عددی: INT، FLOAT، DECIMAL
  • رشته‌ای: CHAR، VARCHAR، TEXT
  • تاریخ و زمان: DATE، TIME، DATETIME
  • بولی: BOOLEAN

عملگرها

  • عملگرهای حسابی: +، -، *، /
  • عملگرهای مقایسه‌ای: =، <>, >، <، >=، <=
  • عملگرهای منطقی: AND، OR، NOT

دستورات DML (زبان دستکاری داده)

  • SELECT: بازیابی داده‌ها از یک یا چند جدول
    SELECT first_name, last_name FROM employees;
  • INSERT: درج رکوردهای جدید در جدول
    INSERT INTO employees (first_name, last_name) VALUES ('علی', 'محمدی');
  • UPDATE: به‌روزرسانی رکوردهای موجود
    UPDATE employees SET salary = 60000 WHERE employee_id = 1;
  • DELETE: حذف رکوردهای موجود
    DELETE FROM employees WHERE employee_id = 1;

دستورات DDL (زبان تعریف داده)

  • CREATE: ایجاد اشیاء جدید مانند جدول
    CREATE TABLE employees (employee_id INT PRIMARY KEY, first_name VARCHAR(50), last_name VARCHAR(50));
  • ALTER: تغییر ساختار جدول
    ALTER TABLE employees ADD email VARCHAR(100);
  • DROP: حذف جدول یا پایگاه داده
    DROP TABLE employees;
  • TRUNCATE: حذف تمام رکوردها از جدول
    TRUNCATE TABLE employees;

دستورات DCL (زبان کنترل داده)

  • GRANT: اعطای دسترسی به کاربران
    GRANT SELECT, INSERT ON employees TO 'user1';
  • REVOKE: لغو دسترسی‌های اعطا شده
    REVOKE SELECT, INSERT ON employees FROM 'user1';

دستورات کنترل تراکنش

  • BEGIN TRANSACTION: شروع یک تراکنش
    BEGIN TRANSACTION;
  • COMMIT: ذخیره دائمی تغییرات
    COMMIT;
  • ROLLBACK: لغو تغییرات
    ROLLBACK;

3. مفاهیم اصلی

جداول، ردیف‌ها و ستون‌ها

جدول در SQL مجموعه‌ای از داده‌ها است که در ردیف‌ها (رکوردها) و ستون‌ها (ویژگی‌ها) سازماندهی شده‌اند. هر ردیف یک موجودیت خاص و هر ستون یک ویژگی از آن موجودیت را نشان می‌دهد.

کلیدهای اصلی و خارجی

  • کلید اصلی (Primary Key): ستونی که هر ردیف را به طور منحصر به فرد شناسایی می‌کند.
    CREATE TABLE employees (employee_id INT PRIMARY KEY, first_name VARCHAR(50));
  • کلید خارجی (Foreign Key): ستونی که به کلید اصلی جدول دیگری اشاره می‌کند.
    CREATE TABLE orders (order_id INT PRIMARY KEY, employee_id INT, FOREIGN KEY (employee_id) REFERENCES employees(employee_id));

محدودیت‌ها (Constraints)

محدودیت‌ها برای تضمین یکپارچگی داده‌ها استفاده می‌شوند:

  • NOT NULL: ستون نمی‌تواند مقدار NULL داشته باشد.
  • UNIQUE: مقادیر ستون باید منحصر به فرد باشند.
  • CHECK: مقادیر ستون باید شرط خاصی را برآورده کنند.
  • DEFAULT: مقدار پیش‌فرض برای ستون.
    CREATE TABLE employees (employee_id INT PRIMARY KEY, salary DECIMAL NOT NULL DEFAULT 50000);

اتصال‌ها (Joins)

اتصال‌ها برای ترکیب داده‌ها از چند جدول استفاده می‌شوند:

  • INNER JOIN: ردیف‌های تطابقی از هر دو جدول.
    SELECT employees.first_name, departments.name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;
  • LEFT JOIN: تمام ردیف‌های جدول چپ و ردیف‌های تطابقی از جدول راست.
  • RIGHT JOIN: تمام ردیف‌های جدول راست و ردیف‌های تطابقی از جدول چپ.
  • FULL JOIN: تمام ردیف‌ها از هر دو جدول.

زیرپرس‌وجوها (Subqueries)

زیرپرس‌وجوها پرس‌وجوهای تودرتو هستند که در بخش‌های مختلف پرس‌وجو استفاده می‌شوند.

SELECT first_name FROM employees WHERE department_id = (SELECT id FROM departments WHERE name = 'فروش');

ایندکس‌ها

ایندکس‌ها سرعت بازیابی داده‌ها را بهبود می‌بخشند.

CREATE INDEX idx_last_name ON employees (last_name);

4. موضوعات پیشرفته

رویه‌های ذخیره‌شده (Stored Procedures)

رویه‌های ذخیره‌شده مجموعه‌ای از دستورات SQL هستند که در پایگاه داده ذخیره شده و با یک فراخوانی اجرا می‌شوند.

CREATE PROCEDURE SelectAllEmployees
AS
SELECT * FROM employees
GO;
EXEC SelectAllEmployees;

تریگرها (Triggers)

تریگرها دستوراتی هستند که به طور خودکار در پاسخ به رویدادهای خاص (مانند INSERT، UPDATE، DELETE) اجرا می‌شوند.

CREATE TRIGGER trg_after_insert
ON employees
FOR INSERT
AS
INSERT INTO audit_table (action, employee_id)
VALUES ('INSERT', NEW.employee_id);

ویوها (Views)

ویوها جداول مجازی هستند که بر اساس نتیجه یک پرس‌وجو ایجاد می‌شوند.

CREATE VIEW vw_employees AS
SELECT employee_id, first_name, last_name
FROM employees
WHERE department_id = 1;

تراکنش‌ها (Transactions)

تراکنش‌ها مجموعه‌ای از عملیات هستند که به عنوان یک واحد اتمی اجرا می‌شوند و از خواص ACID پیروی می‌کنند.

BEGIN TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;
COMMIT;

نرمال‌سازی (Normalization)

نرمال‌سازی فرآیندی است برای کاهش افزونگی داده‌ها و بهبود یکپارچگی. فرم‌های نرمال شامل 1NF، 2NF و 3NF هستند.

تزریق SQL (SQL Injection)

تزریق SQL یک حمله امنیتی است که در آن کد مخرب به پرس‌وجوها وارد می‌شود. برای جلوگیری، از پارامترهای پرس‌وجو استفاده کنید.

-- به جای:
SELECT * FROM users WHERE username = 'user_input';
-- استفاده کنید:
SELECT * FROM users WHERE username = ?;

5. اشکال‌زدایی و بهینه‌سازی

  • بهینه‌سازی پرس‌وجوها: از ایندکس‌ها استفاده کنید، از SELECT * اجتناب کنید و JOINهای مناسب را به کار ببرید.
  • استفاده از EXPLAIN: برای تحلیل نحوه اجرای پرس‌وجو.
    EXPLAIN SELECT * FROM employees WHERE department_id = 1;
  • استراتژی‌های ایندکس‌گذاری: ایندکس‌ها را روی ستون‌های پرکاربرد در WHERE، JOIN و ORDER BY ایجاد کنید.

6. بهترین روش‌ها

  • کد خوانا: از فرمت‌بندی مناسب و نام‌گذاری معنی‌دار استفاده کنید.
  • نام‌گذاری منسجم: برای جداول و ستون‌ها از نام‌های واضح استفاده کنید.
  • **اجتناب از SELECT *** : فقط ستون‌های مورد نیاز را انتخاب کنید.
  • استفاده از تراکنش‌ها: برای عملیات حساس از تراکنش‌ها استفاده کنید.

7. موارد استفاده واقعی

  • عملیات CRUD:
    -- Create
INSERT INTO employees (first_name, last_name) VALUES ('سارا', 'احمدی');
-- Read
SELECT * FROM employees WHERE department_id = 1;
-- Update
UPDATE employees SET salary = 70000 WHERE employee_id = 1;
-- Delete
DELETE FROM employees WHERE employee_id = 1;
  • گزارش‌دهی: جمع‌آوری داده‌ها برای گزارش‌های مدیریتی.
    SELECT department_id, COUNT(*) AS employee_count
FROM employees
GROUP BY department_id;
  • تحلیل داده: استفاده از توابع تجمیعی.
    SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 50000;

8. خطاهای رایج و عیب‌یابی

  • خطاهای نحوی: بررسی دقیق دستورات برای اشتباهات تایپی.
  • خطاهای منطقی: اطمینان از صحت شرایط در WHERE.
  • مسائل عملکرد: بررسی ایندکس‌ها و استفاده از EXPLAIN برای شناسایی گلوگاه‌ها.
