چیت‌شیت جامع جاوااسکریپت (مقدماتی تا پیشرفته)

جاوااسکریپت یک زبان اسکریپتی پویا و چندسکویی (قابل اجرا در مرورگر و سایر محیط‌ها) است که به صفحات وب رفتار تعاملی می‌بخشد. این زبان بر پایه پروتوتایپ (شی‌محور) و استاندارد ECMAScript تعریف شده است. برای استفاده از جاوااسکریپت در مرورگر، معمولاً از تگ HTML <script> استفاده می‌شود. به مثال زیر توجه کنید که عبارت «Hello, JavaScript!» را در کنسول مرورگر نمایش می‌دهد:

<script>
  console.log("Hello, JavaScript!");
</script>

با اجرای این کد، متن «Hello, JavaScript!» در کنسول مرورگر ظاهر می‌شود. حال با مفاهیم پایه شروع می‌کنیم:

مفاهیم پایه و مقدماتی

  • انواع داده (Data Types): جاوااسکریپت دارای تایپ‌دهی پویا است؛ یعنی متغیرها می‌توانند هر نوع داده‌ای بگیرند. انواع داده پایه (Primitive) شامل Undefined، Null، Boolean (صحیح/غلط)، Number (اعداد)، BigInt (اعداد صحیح خیلی بزرگ)، String (رشته‌ها) و Symbol هستند. عملگر typeof نوع داده را برمی‌گرداند (مثلاً typeof 42 // "number"). جدول زیر نمونه‌هایی از داده‌ها و نتیجه typeof متناظر را نشان می‌دهد:

    نوع داده نتیجه typeof
    undefined "undefined"
    null "object"
    true یا false "boolean"
    عدد (مثلاً 123) "number"
    عدد بزرگ (BigInt) "bigint"
    رشته (مثلاً "text") "string"
    نماد (Symbol()) "symbol"
    شیء (مثلاً {}) "object"

    نکته: typeof null به اشتباه "object" برمی‌گردد، لذا برای تشخیص null بهتر است از مقایسه صریح (=== null) استفاده کنید.

  • تعریف متغیرها: سه کلمه‌کلیدی اصلی وجود دارد:

    • var: متغیر تابع-محور یا سراسری تعریف می‌کند. دارای hoisting است (یعنی قبل از تعریف نیز شناخته شده ولی مقدار آن undefined است).
    • let: متغیر بلوک-محور تعریف می‌کند (فقط در بلوک {} جاری معتبر است).
    • const: متغیر بلوک-محور و ثابت تعریف می‌کند؛ پس از مقداردهی اولیه نمی‌توان مقدار آن را تغییر داد.
      مثال:
    var a = 10;    // تابع-محور یا سراسری
let b = 20;    // بلوک-محور
const c = 30;  // بلوک-محور، مقدارش ثابت است

    تفاوت‌های var و let/const: متغیرهای let و const تنها پس از تعریف قابل‌دسترسی هستند (نواحی مرده زمانی دارند) و در بلوک خود محدود می‌مانند. اما var خارج از بلوک نیز در سطح تابع یا گلوبال دیده می‌شود.

  • عبارات و عملگرها: جاوااسکریپت مجموعه‌ای از عملگرها دارد: ариثرметیک (+, -, *, /, %, **)، انتساب (=, +=, -=، …)، مقایسه (==, ===, !=, !==, <, >, <=, >=)، و منطقی (&&, ||, !). مثال استفاده: 

    let x = 5, y = 2;
console.log(x + y);    // 7
console.log(x % y === 0 ? "تقسیم‌پذیر" : "نبستنی"); // "نبستنی"

    نکته حرفه‌ای: همیشه از مقایسه صریح (===, !==) استفاده کنید تا از تبدیل خودکار نوع جلوگیری شود.

  • ساختارهای کنترلی: جاوااسکریپت دستورات شرطی و حلقه‌های استاندارد دارد:

    • if/else: تصمیم‌گیری بر اساس شرط.

      const age = 18;
if (age >= 18) {
  console.log("بزرگسال");
} else {
  console.log("کوچک‌تر از 18");
}

    • switch: بررسی چند شاخه بر اساس یک مقدار:

      const color = "red";
switch (color) {
  case "red": console.log("قرمز"); break;
  case "blue": console.log("آبی"); break;
  default: console.log("نامشخص");
}

    • حلقه‌ها: for (معمولی)، while، do...while. ES6 هم for...of (تکرار روی آرایه یا iterable) و for...in (کلیدهای یک شی) اضافه کرده است: 

      // حلقه for
for (let i = 0; i < 3; i++) {
  console.log(i);
}

// حلقه for...of روی آرایه
for (const item of [10, 20, 30]) {
  console.log(item);
}

    • عبارت ternary: یک عبارت شرطی کوتاه:

      const status = (age >= 18) ? "Adult" : "Minor";

  • توابع (Functions): توابع با کلمه‌کلیدی function یا به‌صورت فلش (arrow) تعریف می‌شوند. مثال تابع ساده:

    function add(x, y) {
  return x + y;
}
const result = add(2, 3); // 5

    • تابع فلش (Arrow Function): سینتکس کوتاه‌تر و بدون بایندینگ خودش برای this و arguments است. مثلاً: 

      const multiply = (a, b) => a * b;
console.log(multiply(3, 4)); // 12

      نکته: توابع فلش نباید به عنوان متد آبجکت استفاده شوند چون this ایستا دارد.

    • پارامترها: می‌توانید پارامتر پیش‌فرض تعریف کنید:

      function greet(name = "Guest") {
  console.log(`Hello, ${name}!`);
}
greet();        // "Hello, Guest!"
greet("Ali");   // "Hello, Ali!"

      همچنین پارامتر REST برای دریافت تعداد متغیر از آرایه استفاده می‌شود:

      function sum(...nums) {
  return nums.reduce((a,b) => a + b, 0);
}
console.log(sum(1, 2, 3)); // 6

    • دامنه (Scope): متغیرهای تعریف‌شده داخل تابع در بیرون از آن در دسترس نیستند. ساختار lexical scope باعث می‌شود تابع درون‌تری به متغیرهای تابع والد دسترسی داشته باشد (قرص‌سازی). در بخش پیشرفته به بستن‌ها (Closures) می‌پردازیم.

  • اشیا و آرایه‌ها:

    • شیء (Object): مجموعه‌ای از جفت‌های کلید-مقدار است. کلیدها معمولاً رشته یا نماد (Symbol) هستند و مقادیر می‌توانند هر نوعی داشته باشند. مثلاً:

      const person = { name: "Ali", age: 25 };
console.log(person.name); // "Ali"
person.job = "Developer";  // اضافه کردن ویژگی جدید

      توجه: اشیا در JS تنها مقادیر متغیر (mutable) هستند.

    • آرایه (Array): نوع خاصی از شیء است که برای لیست مرتب‌شده استفاده می‌شود. آرایه‌ها دارای ویژگی length هستند و از متدهای مفیدی مانند push() (اضافه کردن در انتها) و indexOf() بهره می‌برند. مثال:

      const nums = [1, 2, 3];
nums.push(4);
console.log(nums); // [1, 2, 3, 4]
console.log(nums.indexOf(3)); // 2

    • ساختارهای داده جدید ES6:

      • Map: نگاشت کلید-مقدار که کلیدها می‌توانند هر نوعی باشند.
      • Set: مجموعه‌ای از مقادیر یکتا.
      • WeakMap/WeakSet: نسخه‌های خاصی برای مدیریت حافظه.
        (استفاده از این ساختارها در موقعیت‌های خاص و حرفه‌ای توصیه می‌شود.)

  • عملگرهای رشته‌ای (Template Literals): از بک‌تیک (``) استفاده شده و امکان جاسازی متغیر در رشته با ${expr} را می‌دهد:

    const name = "Ali";
console.log(`Hello, ${name}!`); // "Hello, Ali!"

  • دسترسی به DOM (Front-end): در پروژه‌های وب، با استفاده از جاوااسکریپت می‌توانید به عناصر HTML دسترسی پیدا کنید و آنها را تغییر دهید:

    document.getElementById("myBtn").addEventListener("click", () => {
  document.body.style.backgroundColor = "yellow";
});

    این قطعه کد، با کلیک روی دکمه‌ای به شناسه myBtn، رنگ پس‌زمینه صفحه را تغییر می‌دهد.

  • ذخیره‌سازی محلی (Web Storage):
    متغیرهای محلی در مرورگر:

    localStorage.setItem("user", "Ali");
console.log(localStorage.getItem("user")); // "Ali"

سینتکس و قواعد اصلی

در بالا بخش زیادی از قواعد اولیه ذکر شد. اینجا نکات تکمیلی آورده می‌شود:

  • نیم‌فاصله (Whitespace) و نقطه‌ویرگول: جاوااسکریپت خودبه‌خود انتهای خط را تشخیص می‌دهد، اما بهتر است پس از هر دستور ; قرار داده شود تا خوانایی و ثبات حفظ گردد.

  • کامنت‌ها:

    • خطی: // این یک کامنت است
    • چندخطی: /* توضیح چند‌خطی */

  • حساسیت به حروف: جاوااسکریپت به حروف بزرگ/کوچک حساس است (age با Age متفاوت است).

  • عبارات توصیفی: استفاده از نام متغیر و تابع روشن، کامنت‌های کاربردی و فرمت‌بندی مناسب در کد به خوانایی کمک می‌کند.

مفاهیم پیشرفته

  • بسته‌ها (Closures): ترکیب یک تابع با محیط lexically آن که به تابع اجازه می‌دهد به متغیرهای خارج از محدوده خودش دسترسی داشته باشد. مثال زیر یک بسته ساده را نشان می‌دهد:

    function makeAdder(x) {
  return function(y) {
    return x + y;
  };
}
const add5 = makeAdder(5);
console.log(add5(2)); // 7

    در این مثال، تابع درونی function(y){...} به متغیر x (مقدار ۵) که در تابع بیرونی تعریف شده دسترسی دارد. این یک بسته است.

  • مُدول‌ها (Modules):
    ES6 امکان واردات و صادرات کد میان فایل‌ها را فراهم کرد:

    // File: math.js
export function square(x) { return x * x; }

// File: main.js
import { square } from './math.js';
console.log(square(3)); // 9

    هر ماژول (فایل) می‌تواند توابع یا متغیرهای خود را export کند و از دیگر فایل‌ها import شود. این ساختار به سامان‌دهی کد و بارگذاری بهینه کمک می‌کند.

  • شیوه‌های پیشرفته ارث‌بری:
    جاوااسکریپت بر پایه پروتوتایپ استوار است. هر شیء یک [[Prototype]] دارد که به شیء دیگری اشاره می‌کند (یا null) و به ارث‌بری ویژگی‌ها کمک می‌کند. برای مثال: 

    const parent = { greeting: "Hello" };
const child = Object.create(parent);
console.log(child.greeting); // "Hello" (از والد به ارث رسیده)

    همچنین در ES6 می‌توان از کلاس‌ها استفاده کرد که یک قالب (template) برای ایجاد اشیا هستند و بر مبنای پروتوتایپ عمل می‌کنند:

    class Rectangle {
  constructor(height, width) {
    this.height = height;
    this.width = width;
  }
  area() {
    return this.height * this.width;
  }
}
const r = new Rectangle(5, 3);
console.log(r.area()); // 15

    کلاس‌ها در واقع نوع خاصی از توابع هستند و مانند let/const به صورت Block-scoped رفتار می‌کنند (در ناحیه مرده زمانی قرار دارند و hoisting روی آنها اعمال نمی‌شود).

  • ارجاع‌های خودکار به this: در توابع معمولی، مقدار this بستگی به نحوه فراخوانی تابع دارد. در مثال زیر، در حالت پیش‌فرض و در strict mode، this برابر undefined است: 

    function showThis() {
  console.log(this);
}
showThis(); // در strict mode: undefined

    اما در متدهای یک کلاس یا شیء، this به آن شیء اشاره می‌کند، و در سازنده‌ها (constructor) به شیء جدید. توابع فلش (=>) مقدار this را از محیط لغوی دور برمی‌دارند (غالباً از کد بالاسری)، مثلاً در کد زیر this درون arrow function، همانی است که در محیط بیرونی بوده: 

    class Counter {
  constructor() {
    this.count = 0;
    setInterval(() => {
      this.count++;
      console.log(this.count);
    }, 1000);
  }
}
new Counter(); // this به درستی به شیء Counter اشاره دارد

  • تابع‌های دستکاری داده: ES6 توابع مفیدی را برای آرایه‌ها معرفی کرد:

    • map, filter, reduce, forEach, find, findIndex, some, every، و …
      به عنوان مثال: 
    const nums = [1, 2, 3, 4, 5];
const evens = nums.filter(n => n % 2 === 0);
console.log(evens); // [2, 4]

  • عبارت‌های منظم (RegExp): برای کار با الگوهای متنی استفاده می‌شوند. مثلا /\d+/ الگو برای اعداد، یا استفاده از متدهایی مانند test, exec, match, replace روی رشته‌ها.

  • JSON (JavaScript Object Notation): قالبی سبک برای تبادل داده بر اساس سینتکس اشیا در JS است. جاوااسکریپت دو متد مهم برای کار با JSON دارد:

    const obj = { name: "Ali", age: 25 };
const jsonStr = JSON.stringify(obj);  // تبدیل آبجکت به رشته JSON
const parsed = JSON.parse(jsonStr);   // تبدیل رشته JSON به آبجکت

  • برنامه‌نویسی ناهمگام (Async):

    • Callbacks: در گذشته برای اعمال غیرهمزمان (AJAX, پردازش‌های زمان‌بر) از توابع callback استفاده می‌شد که گاهی منجر به «بهشت پنیر» می‌شد (پیچیدگی زیاد تو در تو).

    • Promises: با ظهور Promise، الگوی جدیدی برای کد غیرهمزمان ارائه شد. یک پرومیس نماینده نتیجه نهایی یا خطای یک عملیات غیربلوکی است. مثال:

      fetch('/api/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

      این کد داده‌های JSON را از سرور می‌گیرد و در صورت خطا آن را در console نمایش می‌دهد.

    • Async/Await: ES8 (ECMAScript 2017) سینتکس async/await را معرفی کرد که روی پرومیس‌ها کار می‌کند و کد را خواناتر می‌کند. مثال معادل بالا با async/await:

      async function loadData() {
  try {
    const res = await fetch('/api/data');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
loadData();

      کد بالا شبیه کد همگام به نظر می‌رسد و برای خطا از try/catch استفاده می‌کند.

ابزارها و کتابخانه‌های مرتبط

  • Node.js: محیط اجرای سمت سرور جاوااسکریپت که امکان اجرای کد JS خارج از مرورگر را فراهم کرده است.

  • مدیر بسته (NPM/Yarn): برای نصب کتابخانه‌ها و مدیریت وابستگی‌ها استفاده می‌شود.

  • ابزارهای ساخت (Build Tools): مانند Webpack، Parcel، Vite برای باندل کردن، تبدیل (transpile) کد ES6 به ES5 با Babel و بهینه‌سازی پروژه.

  • کتابخانه‌ها و فریم‌ورک‌های فرانت‌اند:

    • React: کتابخانه‌ای رایگان و متن‌باز از Meta (فیسبوک) برای ساخت رابط کاربری مبتنی بر کامپوننت. React تمرکز روی UI و رندر مؤلفه‌ها دارد و وضعیت و رندر را بهینه می‌کند.
    • Vue.js: فریم‌ورکی مدرن که می‌توان به صورت تدریجی در پروژه‌های موجود استفاده کرد. Vue ساختار تولید کامپوننت دارد و حتی می‌توان آن را به جای jQuery در وب‌سایت‌های قدیمی قرار داد.
    • Angular: یک فریم‌ورک قوی مبتنی بر TypeScript از گوگل است که برای ساخت اپلیکیشن‌های تک‌صفحه (SPA) استفاده می‌شود.
    • سایر کتابخانه‌ها: jQuery (برای تسهیل دستکاری DOM و AJAX؛ کمتر در پروژه‌های مدرن استفاده می‌شود)، Lodash (ابزار کمک برنامه‌نویسی) و ...

  • ابزارهای توسعه و دیباگ:

    • مرورگرها (Chrome, Firefox) ابزارهای توسعه (DevTools) قدرتمندی دارند برای مشاهده کنسول، breakpoint، پروفایلینگ و …
    • Linters/Formatters: مانند ESLint و Prettier برای حفظ کیفیت و یکنواختی کد.

خطاهای رایج و روش رفع

  • ReferenceError: زمانی ایجاد می‌شود که به متغیری اشاره کنید که تعریف نشده است یا هنوز مقداردهی نشده. مثلاً:

    console.log(nonExistent); // ReferenceError: nonExistent is not defined:contentReference[oaicite:27]{index=27}

  • SyntaxError: خطا در نحو کد (مانند فراموش کردن } یا ;، یا اشتباه در ساختار). با دقت در خطا و تصحیح سینتکس برطرف می‌شود.

  • TypeError: وقتی عملیاتی روی نوع نامناسب انجام شود (مثلاً صدا زدن تابع روی مقداری که تابع نیست یا دسترسی به خصوصیت شیء undefined) رخ می‌دهد. مثلاً:

    let obj = null;
console.log(obj.name); // TypeError: Cannot read properties of null

  • Uncaught Error: خطای پیش‌بینی نشده که باید با try...catch مدیریت شود:

    try {
  JSON.parse("{ bad json }");
} catch (e) {
  console.error("JSON is invalid", e);
}

  • رفع خطا: استفاده از کنسول (console.error)، debugger، breakpoints، و مطالعه پیام خطاها در ابزار توسعه مرورگر کمک می‌کند. فعال کردن Strict mode ("use strict") می‌تواند بسیاری از خطاهای پنهان (مانند استفاده از متغیر تعریف‌نشده) را به استثنا تبدیل کند.

  • ابزارهای کمکی: linters (ESLint) و TypeScript می‌توانند خطاها را پیش از اجرا شناسایی کنند. همچنین StackOverflow و مستندات MDN منابع خوبی برای خطایابی هستند.

الگوهای بهینه‌نویسی و نکات حرفه‌ای

  • استفاده از const پیش‌فرض: اگر مقدار متغیر تغییر نمی‌کند، از const استفاده کنید تا ایمنی بیشتری داشته باشید.
  • پرهیز از تو در تویی غیرضروری: از برجسته شدن (nesting) بیش از حد callbackها یا .then() بپرهیزید. اگر لازم است، توابع و async/await را جدا کنید.
  • کاهش کار روی DOM: تغییرات متعدد روی DOM سنگین است؛ بهتر است تغییرات را دسته‌بندی (fragment) کنید یا از innerHTML یکجا استفاده کنید.
  • محاسبات بهینه: از الگوریتم‌های مناسب و آرایه‌های کم‌هزینه (مثل Map/Set) برای جستجوی سریع استفاده کنید.
  • حافظه و پاکسازی: هنگام کار با رویدادها (Event) و تایمرها (Interval/Timeout)، آنها را در صورت نیاز پاک کنید تا از نشت حافظه جلوگیری شود.
  • استفاده از ابزار پروفایلینگ: مرورگرها امکان پروفایلینگ اجرای کد (Performance) را می‌دهند؛ از آنها برای یافتن گلوگاه (bottleneck) استفاده کنید.
  • الگوهای طراحی (Design Patterns): MVC/MVVM، Singleton، Module pattern و … گاهی در ساختار پروژه‌ها مفیدند. برای مثال ماژول‌ها (IIFE یا ES6 Module) کمک به جداسازی حوزه (namespace) می‌کنند.

استفاده در پروژه‌های واقعی

در پروژه‌های فرانت‌اند معمولاً جاوااسکریپت برای تعامل با کاربر، ارتباط با سرور و مدیریت وضعیت رابط کاربری استفاده می‌شود. چند نمونه کاربردی:

  • ارسال درخواست AJAX با Fetch:

    document.getElementById("submit").addEventListener("click", async (e) => {
  e.preventDefault();
  const data = { name: "Ali", age: 25 };
  try {
    const res = await fetch('/api/user', {
      method: 'POST',
      headers: {'Content-Type': 'application/json'},
      body: JSON.stringify(data)
    });
    const result = await res.json();
    console.log("Server says:", result.message);
  } catch (err) {
    console.error("Network error:", err);
  }
});

    این کد با کلیک روی دکمه فرم، داده‌ها را به سرور می‌فرستد و پاسخ را دریافت و در کنسول چاپ می‌کند.

  • به‌روزرسانی پویا‌ی DOM:

    const items = ["سیب", "موز", "کیوی"];
const listEl = document.querySelector("#myList");
items.forEach(item => {
  const li = document.createElement("li");
  li.textContent = item;
  listEl.appendChild(li);
});

    این کد یک لیست HTML را با داده‌های آرایه پر می‌کند.

  • تعامل با کاربران: افزودن رویدادها (click, input, submit) برای عکس‌العمل به رفتار کاربر:

    const toggleBtn = document.getElementById("toggle");
const content = document.querySelector(".content");
toggleBtn.addEventListener("click", () => {
  content.classList.toggle("hidden");
});

    با این کار می‌توان اجزای صفحه را مخفی/نمایش داد.

  • مثال پروژه کوچک: در پروژه‌های SPA (تک‌صفحه‌ای) معمولاً از چارچوب‌هایی مثل React یا Vue برای مدیریت واسط کاربری و واکنش به تغییرات داده استفاده می‌شود.

منابع یادگیری و نقشه راه پیشنهادی

برای تسلط بر جاوااسکریپت از سطح مقدماتی تا پیشرفته، منابع زیر توصیه می‌شود:

  • مستندات MDN Web Docs: مرجع رسمی جاوااسکریپت و Web APIs با توضیحات جامع و مثال.

  • کتاب‌ها:

    • Eloquent JavaScript (Marijn Haverbeke) – کتاب رایگان آنلاین با عمق بالا.
    • You Don’t Know JS (Kyle Simpson) – مجموعه کتاب‌های متن‌باز برای فهم عمیق زبان.

  • وب‌سایت‌ها و دوره‌های رایگان:

    • JavaScript.info – راهنمای جامع (انگلیسی).
    • freeCodeCamp – تمرین‌های تعاملی و پروژه محور.
    • دوره‌های آموزشی (مثل Udemy، Coursera) با به‌روز‌ترین مباحث.

  • پروژه‌های عملی: شرکت در پروژه‌های متن‌باز یا تمرین با ساخت اپ‌های کوچک (ماشین حساب، TODO List، بازی ساده) مهارت شما را بالا می‌برد.

  • استاندارد ECMAScript: زبان JS به کمک کمیته TC39 در ECMA International تکامل می‌یابد. شما می‌توانید اسناد پیشنهادات جدید (proposals) و نسخه‌های جدید زبان را از سایت tc39.es پیگیری کنید (مثلاً استاندارد ECMAScript 2025 قابلیت‌هایی مثل JSON ماژول و متدهای جدید Set را معرفی کرده است).

  • ابزارها و کدهای نمونه: حتماً با ابزارهای توسعه (کنسول مرورگر، Node.js REPL) تمرین کنید و کدهای آزمایشی بنویسید.

توجه کنید که یادگیری جاوااسکریپت نیاز به تمرین و مطالعه مداوم دارد. این چیت‌شیت مروری سریع است؛ برای درک عمیق‌تر به مستندات و منابع فوق مراجعه کنید. با تمرین مستمر و پروژه‌های واقعی، مهارت شما تکمیل خواهد شد.

منابع: اطلاعات بالا برگرفته از مستندات رسمی و معتبر (MDN، بلاگ‌های برنامه‌نویسی) بوده و به‌طور خاص از منابعی همچون MDN Web Docs استفاده شده است. برای جزئیات بیشتر به این منابع مراجعه کنید.
